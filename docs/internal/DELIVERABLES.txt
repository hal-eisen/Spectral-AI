COMPONENTE 1: DuplScore Optimizer
================================================================================

Archivo: python/dupl_score_optimizer.py (420 líneas)

Descripción:
  Implementa la fórmula de decisión automática para conceptos polisémicos:
  
    DuplScore(C) = (Σ_{s} f(C,s) · R(C,s)) · exp(-γ · D(Sc)) - δ · (|Sc|-1) · size(C)
  
  Decide si un concepto debe DUPLICARSE en múltiples esferas o usar 
  WORMHOLE (puntero O(1)).

Características:
  ✓ Genera vocabulario sintético (21 conceptos, 3 esferas)
  ✓ Calcula DuplScore para conceptos polisémicos
  ✓ Genera decisiones con tabla de análisis
  ✓ Exporta grafo JSON de wormholes
  ✓ Parámetros configurables: γ (gamma), δ (delta), τ (tau)
  ✓ Solo NumPy (sin dependencias externas)

Uso:
  python3 python/dupl_score_optimizer.py \
    --seed 42 \
    --gamma 0.2 \
    --delta 0.001 \
    --tau 0.5 \
    --output wormhole_graph.json

Output:
  ✓ wormhole_graph.json — Grafo de decisiones
  ✓ Tabla console con DuplScore y decisiones
  ✓ Estadísticas de ahorro de memoria

Ejemplo de resultado (4 conceptos polisémicos):
  - bucle: -1.701 DuplScore → WORMHOLE → -2.0KB ahorro
  - energía: -1.713 DuplScore → WORMHOLE → -2.0KB ahorro
  - frecuencia: -4.520 DuplScore → WORMHOLE → -5.0KB ahorro
  - amplitud: -1.727 DuplScore → WORMHOLE → -2.0KB ahorro
  
  Total ahorro: 10.9 KB

COMPONENTE 2: Fuzzy BSH (Fuzzy Bounding Sphere Hierarchy)
================================================================================

Archivo: python/fuzzy_bsh.py (750 líneas)

Descripción:
  Implementa un árbol BSH diferenciable mediante membresía probabilística:
  
    P(token ∈ esfera_k) = softmax(-||token - center_k||² / (2*T²))
  
  Permite entrenar el árbol end-to-end con backpropagation.

Características:
  ✓ Inicialización inteligente desde datos
  ✓ Membresía fuzzy suave controlada por temperatura T
  ✓ Gradient descent analítico (no diferencias finitas)
  ✓ Pérdida espacial multi-objetivo (L_prox + L_cover + L_inter)
  ✓ Simulated annealing: T → 0 para endurecimiento gradual
  ✓ Convergencia rápida: 91.7% accuracy en 200 épocas
  ✓ Solo NumPy

Uso:
  python3 python/fuzzy_bsh.py \
    --num-epochs 200 \
    --seed 42 \
    --learning-rate 0.01 \
    --harden-every 50 \
    --harden-factor 0.9 \
    --output fuzzy_bsh_state.json

Output:
  ✓ fuzzy_bsh_state.json — Estado final (centros, radios, histórico)
  ✓ Tabla console con pérdida y accuracy por época
  ✓ Clustering final discreto al final del training

Ejemplo de resultado (200 épocas, 24 tokens):
  Epoch 0:   T=1.000 L_spatial=1.527 Accuracy=91.7%
  Epoch 100: T=0.810 L_spatial=1.524 Accuracy=91.7%
  Epoch 199: T=0.729 L_spatial=1.533 Accuracy=91.7%
  
  Final Clustering (91.7% accuracy, 22/24 correctos):
  Programación: python, for, while, variable, función, clase, array, import
  Música: ritmo, sample, beat, tempo, acorde, melodía, notas, bucle
  Física: orbita, campo, fuerza, masa, vector, energía, aceleración, frecuencia
